## Test Results

### pure2-hello.cpp2

**Source**:
```cpp2
name: () -> std::string = {
    s: std::string = "world";
    decorate(s);
    return s;
}

decorate: (inout s: std::string) = {
    s = "[" + s + "]";
}

main: () = {
    std::cout << "Hello " << name() << "\n";
}
```

**Cppfort Output**:
```cpp
void decorate(std::string s) {  // ❌ BY-VALUE (should be std::string&)
    {
        (s = (("[" + s) + "]"));
    }
}
```

**Cppfront Reference**:
```cpp
auto decorate(std::string& s) -> void {  // ✓ BY-REFERENCE (correct)
    s = "[" + s + "]";
}
```

**Issue**: `inout` parameter qualifier loses semantics during transpilation.

**Impact**: Function mutations don't propagate - semantic correctness violated.

**Key Differences**:

1. **Parameter passing semantics**:
   - Lost: `inout` → by-reference conversion
   - Generated by-value instead
   - Semantic correctness violation

2. **Function signatures**:
   - Cppfort: `int main()` vs `void decorate(std::string s)`
   - Cppfront: `auto main() -> int` vs `auto decorate(std::string& s) -> void`
   - Consistency issue (trailing return type not always used)

3. **Code structure**:
   - Cppfort adds extra nested blocks: `{ { ... } }`
   - Cppfort adds extra parentheses: `(((a) << b) << c)`
   - Unnecessary noise

4. **Missing runtime support**:
   - Cppfront includes `#include "cpp2util.h"`
   - Cppfort omits (no runtime library)
   - Contracts and safety features unavailable

5. **Source mapping**:
   - Cppfront emits `#line` directives
   - Cppfort omits (debugging harder)

## Blockers Analysis

### 1. Parameter Semantics Loss (CRITICAL)

**Root Cause**: AST→C++ transpilation doesn't preserve parameter qualifier semantics.

**Location**: Likely in parameter declaration transpilation code.

**Fix Required**:
- Map `in` → `const T&`
- Map `out` → `T&` (with write-before-return check)
- Map `inout` → `T&`
- Map `move` → `T&&`
- Map `forward` → template forwarding reference

**Priority**: P0 - Breaks semantic correctness

### 2. Mixed-Mode Parser Support (HIGH)

**Root Cause**: Parser only handles Cpp2 syntax, rejects C++1.

**Location**: Lexer/parser integration.

**Fix Required**:
- Detect C++1 syntax (e.g., `auto name() -> type`)
- Pass through C++1 unchanged
- Only transpile Cpp2 portions

**Priority**: P1 - Blocks 26% of test corpus (50/189 files)

### 3. Semantic Loss Scoring (MEDIUM)

**Root Cause**: Tool couldn't match patterns (possible JSON structure issue).

**Fix Required**:
- Debug why candidate_isomorphs = 0
- Verify JSON structure matches expected format
- Re-run scoring with corrected inputs

**Priority**: P2 - Metrics blocked but tests run

```bash

    --category pure2
```

### Generate Semantic Loss Score
```bash
# 1. Transpile
./build/src/cppfort input.cpp2 /tmp/candidate.cpp

# 2. Generate AST
clang++ -std=c++20 -Xclang -ast-dump -fsyntax-only /tmp/candidate.cpp > /tmp/candidate.ast.txt

# 3. Extract isomorphs
python3 tools/extract_ast_isomorphs.py --ast /tmp/candidate.ast.txt --output /tmp/candidate.isomorph.json

# 4. Tag with MLIR
python3 tools/tag_mlir_regions.py --isomorphs /tmp/candidate.isomorph.json --output /tmp/candidate.tagged.json

# 5. Score
python3 tools/score_semantic_loss.py \
    --reference corpus/tagged/input.tagged.json \
    --candidate /tmp/candidate.tagged.json \
    --output /tmp/loss.json
```
