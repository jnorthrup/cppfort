# Story 1.1: Integrate Rabin-Karp Orbit Scanner

## Status
Approved

---

## Story

**As a** compiler developer,
**I want** Rabin-Karp orbit scanner with simultaneous multi-grammar support,
**so that** cppfort can parse C/C++/CPP2 deterministically in a single pass.

---

## Core Problems (Factored)

### Problem 1: Multi-Resolution Pattern Matching
**Challenge:** Detect patterns of all sizes (1-64 bytes) efficiently
**Solution:** Hierarchical Rabin-Karp with power-of-2 windows (1,2,4,8,16,32,64)
**Benefit:** O(n) single-pass detection, O(1) rolling updates

### Problem 2: Multi-Grammar Ambiguity
**Challenge:** Same syntax valid in C, C++, and CPP2
**Solution:** Simultaneous loading + longest-match-wins + priority tie-breaking
**Benefit:** Natural maximal-munch semantics, handles mixed-mode files

### Problem 3: Context-Dependent Syntax
**Challenge:** `[` means array-index vs lambda-capture vs attribute
**Solution:** Orbit masks track context hosts, validate winners
**Benefit:** Correct disambiguation without backtracking

### Problem 4: Hyper-Parameter Tuning
**Challenge:** What maximum window size?
**Solution:** 99th percentile of successful pattern lengths from corpus analysis
**Benefit:** Covers real patterns without over-allocation

---

## Acceptance Criteria

1. **Hierarchical Rabin-Karp** with power-of-2 windows (1-64 bytes) and O(1) rolling updates
2. **Simultaneous C/C++/CPP2** patterns loaded side-by-side in single scanner
3. **Longest-match-wins** disambiguation (greedy/maximal munch)
4. **Orbit context validation** for confix operators (`{}`, `[]`, `<>`, `()`)
5. **TableGen middle tuple** for pattern specifications
6. **Integration** with `PatternMatcher` and `Machine` infrastructure
7. **Performance** sub-100ms for 10KB files with all grammars active

## Implementation Architecture

### Layer 1: Algorithm Core (Rabin-Karp)

```cpp
class RabinKarp {
    std::array<uint64_t, 7> _hashes;  // Windows: 1,2,4,8,16,32,64 bytes
    size_t _maxLevel;                  // Hyper-parameter

    void initialize(const char* text, size_t len);
    void update(char remove, char add);  // O(1) all levels
    uint64_t hashAt(size_t level) const;
};
```

**Key Properties:**
- 7 simultaneous hash levels (2^0 through 2^6)
- Single rolling update maintains all levels
- Hyper-parameter: maxWindowSize (default 64, tunable to corpus)

### Layer 2: Multi-Grammar Loading

```cpp
class MultiGrammarLoader {
    PatternDatabase _c_patterns;
    PatternDatabase _cpp_patterns;
    PatternDatabase _cpp2_patterns;

    void loadAllGrammars();  // Side-by-side, not sequential
    Match disambiguate(std::vector<Match> candidates);
};
```

**Disambiguation Rules (priority order):**
1. Longest match wins (greedy/maximal munch)
2. Grammar priority: CPP2 > C++ > C (tie-break)
3. Orbit context validation
4. Report ambiguity if all fail

### Layer 3: Orbit Context Tracking

```cpp
enum class OrbitType : uint8_t {
    OpenBrace, CloseBrace, OpenBracket, CloseBracket,
    OpenAngle, CloseAngle, OpenParen, CloseParen
};

class OrbitContext {
    ContextHost _host;  // expr, lambda, decl, template

    bool validates(const Match& pattern) const;
};
```

**Context Hosts:**
- Expression: `[` = array index
- Lambda: `[` = capture list
- Declaration: `[` = attribute
- Template: `<` = template args

### Layer 4: TableGen Bridge

```yaml
# patterns/cpp2_patterns.yaml
patterns:
  - orbit: OpenBracket
    context: LambdaCapture
    target_node: ConstantNode
    grammar: CPP2
    length: 1
```

**Storage:** YAML files per grammar, loaded into `PatternDatabase`

### Layer 5: Integration Points

**Existing Infrastructure:**
- `src/stage0/pattern_matcher.h` - Extend with `matchOrbit()`
- `src/stage0/machine.h` - Add orbit pattern registration
- Sea of Nodes builder - Consume disambiguated patterns

## Key Design Decisions

### Decision 1: Power-of-2 Windows
**Rationale:** Enables efficient hierarchical hashing with binary subdivision
**Trade-off:** Misses non-power-of-2 sizes, but covers 99% of patterns

### Decision 2: Simultaneous Loading
**Rationale:** Real code has mixed C/C++/CPP2, avoids detection overhead
**Trade-off:** 3× pattern database size, but amortized by single scan

### Decision 3: Longest Match Wins
**Rationale:** Maximal munch matches human intuition, provides specificity
**Trade-off:** May be too greedy in edge cases, but deterministic

### Decision 4: Orbit Context Validation
**Rationale:** Same syntax has different meanings in different contexts
**Trade-off:** Requires context tracking, but necessary for correctness

---

## Complexity Analysis

**Time:**
- Initialization: O(k × log(W)) where k=patterns, W=max window
- Per-character: O(log(W)) = O(1) for fixed W (typically 6-7 levels)
- Total scan: O(n × log(W)) ≈ O(n) for n characters

**Space:**
- Hash array: 56 bytes (7 × 8-byte hashes)
- Pattern database: O(p) where p=total patterns across 3 grammars
- Orbit context: O(1) tracking current state
