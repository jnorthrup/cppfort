# Story: Complete Emitter Implementation

## Status: Draft

## Story

As a cpp2 transpiler developer,
I want the emitter to generate complete and correct C++ code,
So that cpp2 programs can be properly compiled without syntax error workarounds.

## Context Source

- Source Document: Brownfield Architecture Analysis (docs/brownfield-architecture.md)
- Enhancement Type: Technical debt resolution
- Existing System Impact: Fixes incomplete emitter that currently has workarounds

## Acceptance Criteria

**Functional Requirements:**

1. Emitter generates valid C++ code for all cpp2 language constructs
2. No syntax error workarounds remain in the emitter code
3. Generated C++ compiles successfully with clang++
4. Output maintains semantic equivalence to input cpp2

**Integration Requirements:**

5. Existing regression tests continue to pass
6. Build system integration remains functional
7. No breaking changes to emitter API

**Quality Requirements:**

8. Code follows project coding standards
9. Emitter handles edge cases properly
10. Error messages are clear and helpful

## Dev Technical Guidance

### Existing System Context

Based on brownfield analysis, the emitter (stage0) has incomplete implementations and workarounds for syntax errors. The emitter is located in `src/stage0/emitter*` files and handles AST-to-C++ code generation.

### Integration Approach

- Work within existing emitter framework
- Maintain compatibility with current AST structure
- Ensure generated C++ uses proper clang++ syntax
- Follow existing code generation patterns

### Technical Constraints

- Must work with current AST node classes (some incomplete)
- Cannot break existing build system
- Must maintain LLVM/MLIR integration compatibility
- Follow Sea of Nodes IR principles where applicable

### Key Files to Modify

- `src/stage0/emitter.cpp` - Main emitter implementation
- `src/stage0/emit_*` files - Specific emission helpers
- `include/cpp2_impl.h` - AST node definitions

### Testing Requirements

- Run full regression test suite after changes
- Validate generated C++ compiles with clang++
- Test edge cases and error conditions
- Ensure no performance regressions
