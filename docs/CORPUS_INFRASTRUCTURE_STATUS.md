### Phase 5: Semantic Loss Scoring Framework ✓

**Implemented scoring tools**:
- `tools/score_semantic_loss.py`: Compares reference vs candidate ASTs
- `tools/aggregate_corpus_scores.py`: Generates corpus-wide report

**Loss metrics implemented**:
- Structural distance: Normalized graph edit distance
- Type distance: Type inference mismatches
- Operation distance: MLIR operation differences
- Combined loss: Weighted sum (50% structural + 30% type + 20% operation)

**Loss categories**:
- 0.0: Zero loss (perfect match)
- 0.0-0.1: Low loss (minor syntactic differences)
- 0.1-0.5: Medium loss (different code generation strategies)
- >0.5: High loss (semantic differences)

```bash
    ./build/src/cppfort "$cpp2" "corpus/candidate/$(basename $cpp2 .cpp2).cpp"
done

# Generate candidate AST dumps
for cpp in corpus/candidate/*.cpp; do
    clang++ -std=c++20 -Xclang -ast-dump -fsyntax-only "$cpp" > "${cpp%.cpp}.ast.txt"
done

# Extract candidate isomorphs
for ast in corpus/candidate/*.ast.txt; do
    ./tools/extract_ast_isomorphs.py --ast "$ast" --output "corpus/candidate/$(basename $ast .ast.txt).isomorph.json"
done
```

### 3. Run Semantic Loss Scoring

Compare candidates against reference:
```bash

mkdir -p corpus/scores
for ref in corpus/tagged/*.tagged.json; do
    basename=$(basename $ref .tagged.json)
    ./tools/score_semantic_loss.py \
        --reference "$ref" \
        --candidate "corpus/candidate/$basename.isomorph.json" \
        --output "corpus/scores/$basename.loss.json"
done

# Generate corpus-wide report
./tools/aggregate_corpus_scores.py \
    --scores-dir corpus/scores \
    --output corpus/CORPUS_LOSS_REPORT.md
```
